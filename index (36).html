🗄️ Configuração do Supabase

Siga o passo a passo abaixo para conectar o painel administrativo ao banco de dados Supabase (grátis). Assim, todas as alterações feitas aqui serão salvas online e ficarão disponíveis em qualquer dispositivo.

📋 Passo a Passo
  1. Acesse supabase.com e crie uma conta gratuita (pode logar com GitHub ou Google).
  2. Clique em "New Project", dê um nome (ex: lsk-chocolates), defina uma senha forte para o banco e escolha a região South America (São Paulo).
  3. Aguarde o projeto ser criado (~2 min). Quando estiver pronto, vá em SQL Editor (ícone de tabela no menu lateral).
  4. Clique em "New query", cole o bloco de SQL abaixo e clique em "Run".
  5. Vá em Storage (menu lateral) → "New bucket" → nome: lsk-images → marque "Public bucket" → Create.
  6. Em Storage → lsk-images → "Policies" → "New policy" → "For full customization" e cole as 4 políticas do bloco 2 do SQL abaixo.
  7. Volte em Project Settings (ícone de engrenagem) → API. Copie a Project URL e a chave anon public.
  8. Volte aqui no painel e cole os valores no campo abaixo (eles ficam salvos apenas no seu navegador).
  9. Pronto! Clique em "Conectar ao Supabase". O indicador no topo vai ficar verde.
🔑 Credenciais (salvas no navegador)

Cole aqui a URL e a chave anon do seu projeto Supabase.

✓ Configurações salvas com sucesso!

📜 BLOCO 1 — Criar as tabelas (rode no SQL Editor)
-- ============================================================
-- LSK CHOCOLATES - SETUP COMPLETO DO BANCO
-- Rode este script UMA VEZ no SQL Editor do Supabase
-- ============================================================

-- 1) Tabela de produtos
create table if not exists products (
  id bigserial primary key,
  title text not null,
  category text not null,
  description text,
  price numeric(10,2) not null default 0,
  stock int default 0,
  mp_link text,
  image text,
  active boolean default true,
  created_at timestamp default now()
);

-- 2) Tabela de slides da barra superior (carrossel do topo)
create table if not exists topo_slides (
  id bigserial primary key,
  icon text default '✨',
  text text not null,
  link text,
  order_index int default 0,
  created_at timestamp default now()
);

-- 3) Tabela de slides do carrossel do hero (Fazenda)
create table if not exists fazenda_slides (
  id bigserial primary key,
  title text not null,
  text text,
  image text,
  order_index int default 0,
  created_at timestamp default now()
);

-- 4) Tabela de avisos (Quadro de Avisos)
create table if not exists avisos (
  id bigserial primary key,
  title text not null,
  text text,
  image text,
  category text default 'Geral',
  pinned boolean default false,
  created_at timestamp default now()
);

-- 5) Tabela de configurações da loja (WhatsApp, Instagram, etc.)
create table if not exists store_config (
  id int primary key default 1,
  whatsapp text,
  instagram text,
  email text,
  mp_public_key text,
  mp_access_token text,
  updated_at timestamp default now()
);

-- Insere registro padrão de configuração (id=1)
insert into store_config (id, whatsapp, instagram, email)
values (1, '5511999999999', 'https://instagram.com/lskchocolates', 'contato@lskchocolates.com.br')
on conflict (id) do nothing;

-- 6) Tabela de regras de frete
create table if not exists frete_config (
  id int primary key default 1,
  base numeric(10,2) default 25.00,
  gratis_prefixos text default '04,08,13',
  adicional_norte numeric(10,2) default 35.00,
  adicional_nordeste numeric(10,2) default 30.00,
  adicional_sul numeric(10,2) default 20.00,
  adicional_centro_oeste numeric(10,2) default 28.00,
  updated_at timestamp default now()
);

-- Insere registro padrão de frete (id=1)
insert into store_config (id) values (1) on conflict do nothing;
insert into frete_config (id) values (1) on conflict do nothing;

-- 7) Habilita acesso público de leitura e escrita (RLS liberado)
-- IMPORTANTE: isso libera acesso anônimo total. Ideal para uso em single-page
-- sem backend. Para produção real, implemente autenticação.
alter table products       enable row level security;
alter table topo_slides    enable row level security;
alter table fazenda_slides enable row level security;
alter table avisos         enable row level security;
alter table store_config   enable row level security;
alter table frete_config   enable row level security;

drop policy if exists "Public all products"       on products;
drop policy if exists "Public all topo_slides"    on topo_slides;
drop policy if exists "Public all fazenda_slides" on fazenda_slides;
drop policy if exists "Public all avisos"         on avisos;
drop policy if exists "Public all store_config"   on store_config;
drop policy if exists "Public all frete_config"   on frete_config;

create policy "Public all products"       on products       for all using (true) with check (true);
create policy "Public all topo_slides"    on topo_slides    for all using (true) with check (true);
create policy "Public all fazenda_slides" on fazenda_slides for all using (true) with check (true);
create policy "Public all avisos"         on avisos         for all using (true) with check (true);
create policy "Public all store_config"   on store_config   for all using (true) with check (true);
create policy "Public all frete_config"   on frete_config   for all using (true) with check (true);

-- Pronto! Você pode testar voltando no painel e clicando em "Testar Conexão".
